# Meeting Notes — Agent Instructions

`TEMPLATE.md` is the canonical template for all meeting notes in this folder. Follow these instructions to produce meeting notes from it.

## File Naming

Save each meeting's notes as `YYYY-MM-DD.md` using the date of the meeting (e.g. `2026-03-17.md`).

## Workflow

1. Copy `TEMPLATE.md` into a new file named with the meeting date.
2. Replace `YYYY-MM-DD` in the heading with the actual date.
3. Fill in the **Resources** section with links to the AI summary, transcript, and recording.
4. Populate **Topics Discussed** with the subjects covered in the meeting.
5. List any decisions made under **Decisions**. If none, write "None."
6. List any deliverables identified under **Deliverables**. If none, write "None."
7. Fill in the **Action Items** table with owner, task description, and due date for each item.
8. Add a row to the table in `README.md` for the new meeting. Insert the row at the top of the table (below the header) so that notes remain in descending date order. The row should contain the date, a link labeled "Full notes" pointing to the notes file, and a 1-2 sentence summary of the meeting's key topics.

## Style Guidance

- Use concise bullet points, not full paragraphs.
- Keep topic descriptions to 1-2 sentences.
- Attribute action items to specific people when possible.

## Sections With No Content

Keep the section heading and write "None." rather than removing the section. This keeps the structure consistent across meetings.

## Communication Style

Apply these guidelines to all writing in this repository, including documents, commit messages, pull request descriptions, and responses to users:

- Do not use contrastive language.
- Do not line break with horizontal rules.
- Don't be a sycophant.
- Be direct, but don't use partial clauses in your sentence structure.
- Use complete thoughts that flow naturally.
- Provide a neutral voice that's both playful but professional at the same time.
